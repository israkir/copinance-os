# API Reference

Reference for interfaces and extension points when integrating or extending Copinance OS.

## Data providers

Implement these interfaces to plug in custom market, fundamental, alternative, or macro data sources:

- [Data Provider Interfaces](data-providers) — `MarketDataProvider`, `FundamentalDataProvider`, `AlternativeDataProvider`, `MacroeconomicDataProvider`. Built-in **SEC/EDGAR** support uses **`EdgarToolsFundamentalProvider`** (`data.providers.sec.edgartools`).

Implementation guide: [Extending](/developer-guide/extending).

## Orchestration and use cases

- **`ResearchOrchestrator`** — `copinance_os.core.orchestrator.research_orchestrator`; use `container.research_orchestrator()` for `run_job`. **`DefaultJobRunner`** dispatches **`Job`** instances to **`AnalysisExecutor`** implementations. See [Architecture](/developer-guide/architecture).
- **Use cases and requests** — `copinance_os.research.workflows.*` (analyze, market, profile, fundamentals); see [Using as a Library](/getting-started/library#library-api-reference).

## LLM providers

- **`LLMProvider`** (`copinance_os.ai.llm.providers.base`) — **`generate_text`**, **`generate_with_tools`**, **`generate_text_stream`** (async iteration of **`LLMTextStreamEvent`**). Factories: **`LLMProviderFactory`**, config: **`LLMConfig`** (`text_streaming_mode`, **`provider_config["disable_native_text_stream"]`**). **Gemini** and **Ollama** support native streaming in **`auto`** mode with buffered fallback.
- Details: [Using as a Library — LLM text streaming](/getting-started/library#llm-text-streaming-library--llmprovider).

## Other extension points

- **Architecture and ports**: [Architecture](/developer-guide/architecture) — domain ports (repositories, `JobRunner`, analysis executors, tools, storage).
- **Container**: [Using as a Library](/getting-started/library#library-api-reference) — `get_container()` entry points, request/response types, domain types.
