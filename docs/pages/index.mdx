# Copinance OS

**Copinance OS** is a Python **financial research operating system**: a library and CLI for deterministic, data-grounded market and macro analysis, plus optional **question-driven** workflows where an LLM **uses tools** to explain and synthesize—not to invent numbers.

The product vision is in the [Manifesto](https://github.com/copinance/copinance-os/blob/main/MANIFESTO.md) on the repository: research that meets people at their literacy level, with clear assumptions and traceable methodology.

## What this system is

| Idea | What it means in practice |
|------|---------------------------|
| **Deterministic core** | Quotes, history, fundamentals, regime tools, macro series, and QuantLib BSM Greeks are computed in **domain + data + core** layers. Results are structured (Pydantic models), not ad hoc prose. |
| **AI as an explanation layer** | LLMs summarize, compare, and answer natural-language questions **using registered tools** that fetch or compute from real providers. They are not the source of truth for prices, Greeks, or macro prints. |
| **Orchestration first** | Analysis runs flow through **`ResearchOrchestrator`** (and the **research workflows** built on it). Custom **JobRunner** implementations remain available for queues and advanced routing. |
| **Pluggable boundaries** | **Ports** in `copinance_os.domain.ports` define data providers, tools, repositories, executors, and strategies; **infra** wires implementations via dependency injection. |

## Repository layout (mental model)

```
copinance_os/
  domain/              # Models, ports, strategies (pure contracts; no I/O)
  research/workflows/  # Use cases: analyze, market, profile, fundamentals, backtest
  core/                # Orchestrator, execution engine, pipeline tools
  data/                # Providers, cache, repositories, loaders, analytics (QuantLib, etc.)
  ai/llm/              # LLM providers, prompts (explanation / agent loop)
  infra/               # Settings, logging, DI container
  interfaces/cli/      # Typer CLI
```

## Core concepts

- **Analysis profile** — Context for a run (financial literacy, preferences). Not a user account; your application owns identity if you need it.
- **Deterministic analysis** — Fixed pipelines (instrument, options, macro/regime) with explicit steps and structured outputs.
- **Question-driven analysis** — You ask a question; the model selects **tools** (market data, fundamentals, regime, macro) and returns an answer grounded in tool outputs.
- **Research orchestration** — Prefer `ResearchOrchestrator.run_job` (via the DI container) over calling executor internals directly; see the [architecture guide](developer-guide/architecture).
