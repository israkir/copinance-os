# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),

## Release Strategy

**Note:** This project is currently in active development with frequent architectural changes and improvements. As such, no package releases will be made until the project reaches a mature and stable state. All changes are tracked in the `[Unreleased]` section below. Once the project stabilizes, versioned releases will begin following Semantic Versioning.

## [Unreleased]

### Added

- **Multi-turn question-driven analysis (library)**: `AnalyzeInstrumentRequest` and `AnalyzeMarketRequest` accept **`conversation_history`** (`list[LLMConversationTurn]`: alternating user/assistant, ending with assistant). The new user message is **`question`**. Prior turns are passed to **Gemini** as native `Content` turns + **systemInstruction**, and to **Ollama** via **`/api/chat`** `messages` — not concatenated into the shared prompt template. On success, executor results may include **`conversation_turns`** for the next call. **`parse_conversation_history`** (job context dicts) validates pairs. Default runners omit `conversation_history` from context when empty. **CLI:** single-turn only (no conversation flags); use the API for multi-turn.
- **LLM text streaming**: **`LLMProvider.generate_text_stream`** with **`LLMTextStreamEvent`** (Pydantic: **`text_delta`**, **`done`**, **`error`**, **`rollback`**), modes **`auto`** / **`native`** / **`buffered`**, **`LLMConfig.text_streaming_mode`**, and **`provider_config["disable_native_text_stream"]`**. Helpers **`ai/llm/tool_loop_streaming.py`**; native paths for **Gemini** (`generate_content_stream`) and **Ollama** (NDJSON **`/api/generate`**). Question-driven **`generate_with_tools`** accepts **`stream`** and **`on_stream_event`**; **`rollback`** when a streamed turn contained tool calls. Default stdout handler **`core/execution_engine/llm_stream_stdout.py`**; executor sets **`analysis_streamed`** and passes job **`context["stream"]`**. **CLI:** **`--stream`** on the **`analyze`** group (before **`equity` / `options` / `macro`**) and at **root** generic research (**`parse_root_argv`**); **`--json`** disables streaming. **`AnalyzeInstrumentRequest` / `AnalyzeMarketRequest`**: **`stream`**. **`DefaultAnalyze*Runner`**: forwards **`stream`** in context. Unit tests: argv dispatch, runners, workflows.
- **CLI — package layout**: Modular layout under `interfaces/cli/`: `commands/` (analyze, cache, market, profile, generic research), `shared/` (utils, formatting, error handling, profile context, market helpers, run-job JSON output), `dispatch/argv.py` (natural-language vs Typer routing), `main.py` (`main()` entry), `root.py` (Typer apps). Lazy **`app`** / **`version_app`** via `interfaces/cli/__init__.py` **`__getattr__`** for fast imports. **`[project.scripts]`** entry: **`copinance = "copinance_os.interfaces.cli:main"`**.
- **Question-driven execution — tool summary**: `core/execution_engine/question_driven_tool_summary.py` — **`is_tool_call_json_text`**, **`summarize_tool_calls_for_display`**, **`build_partial_synthesis_message`** for human-readable fallbacks when the final LLM text is missing or still looks like a tool call.
- **Analysis reports**: **`build_question_driven_analysis_report`** in **`domain/services/question_driven_analysis_report.py`** maps question-driven executor output to **`AnalysisReport`**; **`RunJobResult`** gains optional **`report_exclusion_reason`** (`ReportExclusionReason.UNKNOWN_EXECUTOR_TYPE`) when results exist but no report builder is registered for `execution_type`.
- **CLI — machine-readable output**: **`--json`** on the **`analyze`** and **`market`** Typer groups (before the subcommand) prints JSON to stdout: **`analyze`** emits **`RunJobResult.model_dump(mode="json")`** (including **`report`** and **`report_exclusion_reason`**); **`market`** emits a small envelope per subcommand (`command`, `response` / `rows` / `chain`, etc.). Helpers **`print_run_job_result_json`** and **`print_json_stdout`** in **`interfaces/cli/shared/utils.py`** (run-job wiring in **`shared/run_job_output.py`**).
- **Market data coercion**: **`CoerceMarketDataPointsResult`**, **`coerce_sorted_market_data_points_detailed`**, skip **warnings** with counts, and **`strict=True`** on **`coerce_sorted_market_data_points`** for fail-fast validation. Documentation in [API Reference — Data providers](/api-reference/data-providers#market-data-point-coercion) and User Guide CLI.
- **SEC / EDGAR**: **`edgartools`** dependency and **`EdgarToolsFundamentalProvider`** in `copinance_os.data.providers.sec.edgartools`; DI **`sec_filings_provider`** for question-driven agent tools (`get_sec_filings`, `get_sec_filing_content`). SEC identity via **`EDGAR_IDENTITY`** / **`COPINANCEOS_EDGAR_IDENTITY`** (defaults in settings). **Cache** integration with per-entry TTL via `CacheManager` to limit EDGAR traffic. **Pipeline tools** (registry + `fundamental_data.py`): company EDGAR profile, company facts statement, compare financials, XBRL statement table, insider Form 4, 13F institutional holdings, registered-fund search/entity/filings/portfolio/latest report — analytical tools ordered before generic filing list/content for LLM prompts.
- **Container/DI**: Optional `storage_type` and `storage_path` on `get_container()` for library integrators; pass `storage_type="memory"` to avoid creating a `.copinance` directory on disk without using `COPINANCEOS_STORAGE_TYPE`. Unit tests for container storage overrides.
- **Market search**: `EXCHANGE_DISPLAY_NAMES` and `format_exchange()` for human-readable market names (e.g. NMS → NASDAQ, NYQ → NYSE); "Market" column in search results; `longName`, `shortName`, and `exch_disp` from yfinance in results.
- **Container/DI**: Lazy container proxy so no container is created at import time; first `get_container()` or attribute access wins. Unit tests for container cache configuration.

### Changed

- **Documentation — site Introduction (`docs/pages/index.mdx`)**: Landing page refocused on **conceptual clarity**—what the system is for, how layers differ, deterministic vs question-driven modes, orchestration, tools, and configuration—without leaning on class names on the intro; **Core concepts** expanded then rewritten in plain language; **Capabilities** and **What this system is** refreshed; codebase organization described as areas of responsibility with a pointer to the full package tree in Architecture.
- **Documentation — README (`Using the CLI`)**: Examples unified on the **`copinance`** command after install (`pip install -e .` / `make setup`); **Verify install** via `copinance version`; optional one-line note that `python -m copinance_os.interfaces.cli` is equivalent; consolidated command blocks for natural language, profiles, market, analysis, macro, JSON, streaming, cache, completion, and machine-readable output; removed duplicate `python -m` sample invocations.
- **CLI — streamed question-driven output**: **`render_run_job_results`** always prints the **Analysis** Rich panel (markdown / tables) when **`analysis`** is present, even if **`analysis_streamed`** is true, so users always get a readable copy-paste block after live tokens; dim hint explains live vs panel. User guide CLI updated accordingly.
- **CLI — help text for streaming**: Root **`typer.Typer` help** and **`_root_cli_epilog_natural_language`** (`--stream` example); lazy **`analyze`** command help mentions group **`--json`** / **`--stream`**; **`analyze_app` help** and **`--stream` Option** text; **`dispatch/argv.py`** docstring and stderr examples.
- **LLM providers (Gemini, Ollama)**: Question-driven tool-calling loops set **`synthesis_status`** (**`complete`** vs **`partial`**) and optional **`llm_synthesis_error`**; on partial completion (e.g. iteration error after tools ran, or final text still resembling JSON tool calls), substitute **`build_partial_synthesis_message`** so users see a deterministic tool summary instead of raw tool JSON.
- **CLI**: Removed monolithic **`interfaces/cli/*.py`** command modules at package root in favor of **`commands/`**, **`shared/`**, **`dispatch/`**, **`main.py`**, and **`root.py`**. Ruff **`PLC0415`** per-file ignores for lazy imports in **`interfaces/cli/__init__.py`** and **`interfaces/cli/main.py`**.
- **CLI / user guide**: Root `copinance --help` `market` summary aligned with the `market` Typer app (“options (BSM Greeks via QuantLib)”); `market options --expiration` Typer help aligned with the user guide default wording. CLI reference table (`market`, `cache`) and macro `--timeframe` default (`mid_term`) clarified.
- **Contributing**: `pip install -e ".[dev]"` only — removed nonexistent `.[dev,docs]` extra from the setup example.
- **Yahoo options:** Default `--expiration` now picks the earliest listed expiry **on or after today** (not raw `ticker.options[0]`), avoiding expired front months and missing BSM Greeks the day after expiry.
- **BSM / QuantLib:** Treat expiry **strictly before** evaluation as expired (`maturity < eval`); allow **same calendar day** as 0DTE. With normalized IV (see below), Greeks can compute for live expiries.
- **Yahoo options / BSM:** Normalize option-chain `impliedVolatility` when value is **> 1** (treat as percent points → divide by 100) so QuantLib BSM Greeks match σ as a fraction; fixes `-` Greeks when Yahoo returns values like `13.67` for ~13.7% vol.
- **CLI — `market options`:** Always shows Delta/Gamma/Theta/Vega/Rho columns by default (cells `-` when Greeks are not computed); `--no-greeks` hides them. Expanded help text and short flags for expiration/side. Docs (user guide, README, library API, options chain metadata) updated for BSM Greeks and `COPINANCEOS_OPTION_GREEKS_*` settings.
- **Documentation** (README, `CHANGELOG`, docs site, user guide, library/API reference): multi-turn question-driven analysis (`conversation_history`, provider-native chat history); root natural-language CLI (`copinance "…"`), `main`/`dispatch` vs lazy Typer, executor/`DefaultJobRunner`; `.venv` / `make setup`; storage vs cache and `storage_type="memory"`; JSON CLI output, structured reports, and market point coercion; package metadata and architecture/extension docs.
- **Market search**: Cache hits that are all "stub" instruments (e.g. bad symbol "APPLE") are treated as empty so the provider is queried and can return real results (e.g. AAPL). When symbol lookup fails, fall back to provider name search (yfinance Search) so queries like "APPLE" resolve to AAPL.
- **Container/DI**: `cache_enabled` and `cache_manager` overrides are applied when returning an existing container, so library callers (e.g. `get_container(cache_enabled=False)`) are no longer ignored.
